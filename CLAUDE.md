## Project Overview

**simple-blog** is a personal website/blog built with:
- Next.js 16 (App Router) + React 19
- TypeScript 5.9 (strict mode)
- Tailwind CSS v4 with custom dark mode
- shadcn/ui (Radix UI primitives + CVA)
- MDX blog posts with syntax highlighting (rehype-pretty-code + shiki)

The site uses server components by default, static generation for pages, and stores blog content as Markdown files with frontmatter in `src/blogs/`.

## Common Development Commands

From `package.json`:

```bash
npm run dev        # Start dev server with Turbopack
npm run build      # Production build
npm run start      # Start production server
npm run lint       # Run Biome linter (check only)
npm run lint:fix   # Run Biome with auto-fixes
npm run prepare    # Set up Huski
</code></pre>

**Testing:** No testing framework is currently configured.

## Architecture & Structure

### Key Directories
- `src/app/` - Next.js App Router pages (server components by default)
  - `blog/[slug]/` - Dynamic blog post routes with `generateStaticParams`
  - `blog/page.tsx` - Blog listing with tag filtering
  - `projects/page.tsx` - Projects page (currently "under maintenance")
  - `layout.tsx` - Root layout with theme provider + navbar
- `src/components/`
  - `ui/` - shadcn/ui components (button, input, label, etc.)
  - Layout components: Navbar, Footer, Header, ThemeToggle
  - Blog components: BlogItem, AllBlogs, TopBlogs
  - Project/Experience components
- `src/lib/`
  - `blogs.ts` - Reads markdown files, parses frontmatter, fetches blog data
  - `mdx.ts` - MDX compilation with plugins (rehype-slug, rehype-pretty-code, remark-gfm)
  - `utils.ts` - `cn()` utility for Tailwind class merging
- `src/constants/index.ts` - Centralized data: technologies, experience, projects, links
- `src/types/index.ts` - TypeScript type definitions
- `src/blogs/` - Markdown blog posts (20+) with frontmatter
- `src/assets/` - Static images (company logos, tech stack icons)
- `public/` - Generated files: `sitemap.xml`, `robots.txt`, `rss.xml`, images

### Content Flow
1. Blog posts are `.md` files in `src/blogs/` with frontmatter:
   `title`, `datePublished`, `seoTitle`, `seoDescription`, `tags`, `notebookLM` (optional)
2. `src/lib/blogs.ts` reads files, parses frontmatter with `gray-matter`
3. `src/app/blog/page.tsx` lists all posts (server component, static generation)
4. `src/app/blog/[slug]/page.tsx` renders individual posts using `next-mdx-remote`
5. MDX compiled with `src/lib/mdx.ts` - includes:
   - `rehype-slug` + `rehype-autolink-headings` for heading anchors
   - `rehype-pretty-code` with `shiki` for syntax highlighting
   - `remark-gfm` for GitHub-flavored markdown
6. RSS feed generated in `src/app/blog/page.tsx` via `generateStaticParams`
7. Sitemap generated post-build via `next-sitemap`

### Styling & Theme
- Tailwind CSS v4 with PostCSS (`@tailwindcss/postcss`)
- Dark mode via CSS custom properties + `next-themes`
- Global styles in `src/app/globals.css` (includes prose styles for blog content)
- UI components use `class-variance-authority` for variants
- Icon library: `lucide-react`

### Configuration Files to Know
- `tsconfig.json` - Path alias: `@/*` → `./src/*`; strict mode enabled
- `biome.json` - Linter/formatter: tabs (4), double quotes, semicolons always
- `next.config.ts` - Minimal (default)
- `components.json` - shadcn/ui config (New York style, RSC enabled, Tailwind v4)
- `next-sitemap.config.js` - Uses `SITE_URL` env var, generates robots.txt
- `.husky/` - Pre-commit hook runs Biome on staged files via lint-staged

### Environment Variables
- `NEXT_PUBLIC_GA_ID` - Google Analytics ID (used by `@next/third-parties`)
- `SITE_URL` - Site URL for sitemap/RSS generation
- See `.env.example` for reference

### Build Output
- `.next/` - Next.js build output
- `public/sitemap.xml`, `public/robots.txt` - Generated by `next-sitemap` (postbuild)
- `public/rss.xml` - Generated during blog build

### Package Manager
- Bun lockfile present (`bun.lock`), but project works with npm
- `trustedDependencies` includes Biome and Tailwind CSS oxide

### Recent Patterns
- Prefers server components, moves client components to dedicated files when needed
- Uses TypeScript strictly; avoid `any` types
- Follows Biome formatting (tabs, double quotes, semicolons)
- Blog content lives in `src/blogs/` as markdown - do not move to database
